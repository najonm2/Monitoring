## BACKUP SLIDE 1: Technical Architecture

### Title: "Enterprise-Grade Architecture"

**System Components:**
```
User Browser (HTTPS)
    ↓
Windows IIS + SSO Authentication
    ↓
Django Application
    ├─ URL Routing
    ├─ Views & API
    ├─ Cache Layer (2-min TTL)
    └─ Business Logic
    ↓
Oracle Client (LOB Reading, 10MB)
    ↓
Oracle Databases
    ├─ Level3 (INFA_PCREPO)
    └─ MAPDQPRD (MDM/ERP/ADF)
```

**Technology Stack:**
- Backend: Django 6.0 (Python 3.11)
- Database: Oracle 19c with python-oracledb
- Caching: In-memory (120-sec TTL)
- Security: Windows SSO, HTTPS/TLS, RBAC
- Deployment: Windows IIS (Production)

---

## BACKUP SLIDE 2: Security & Compliance

### Title: "Enterprise Security Standards"

**Multi-Layer Security:**

**Layer 1: Network**
- Corporate VPN required
- Firewall restrictions (internal IPs only)
- Database IP whitelisting

**Layer 2: Authentication**
- Windows SSO (Single Sign-On)
- REMOTE_USER environment variable
- Role-based access control (RBAC)

**Layer 3: Transport**
- HTTPS/TLS 1.2+ encryption
- Secure cookies (HttpOnly, Secure flags)
- CSRF protection

**Layer 4: Database**
- Read-only database users
- Oracle Wallet for credentials
- Parameterized queries (SQL injection prevention)

**Layer 5: Application**
- XSS protection headers
- Clickjacking protection
- Content Security Policy (CSP)

**Layer 6: Audit**
- Complete access logging
- Error tracking and alerting
- Compliance audit trail
